프롬프트 엔지니어링이란?

AI와 효과적으로 대화하기 위한 지시문 작성 기술입니다. 명확하고 구체적인 프롬프트는 더 정확한 결과를 생성합니다.

💡 핵심 원칙
  • 역할 지시 (System Message): "너는 경제 기사 요약 전문가야"
  • 구체적인 형식 요구: 답변의 어조, 길이, 구조 명시
  • Few-shot Learning: 예시를 통한 패턴 학습

실전 예시

❌ 나쁜 예시

"마케팅 전략 알려줘"

✅ 좋은 예시

"너는 10년 경력의 디지털 마케팅 전문가야. 30대 여성을 타겟으로 한 친환경 화장품 브랜드의 인스타그램 마케팅 전략을 3가지 제시해줘. 각 전략은 예산, 기대효과, 실행방법을 포함해야 해."

지침 (Instructions)

GPT의 성격과 행동을 결정하는 가장 중요한 요소

🎯 지침이란?

지침(Instructions)은 GPT가 어떻게 행동해야 하는지 알려주는 사용 설명서입니다. 마치 사람에게 업무 매뉴얼을 주는 것처럼, GPT에게 역할과 규칙을 알려줍니다.

📝 초급자를 위한 설명

Q: 왜 필요한가요?

A: 지침이 없으면 GPT는 일반적인 ChatGPT처럼 행동합니다. 하지만 지침을 설정하면 "고객 상담 전문가", "영어 회화 선생님", "법률 자문 봇" 등 특정 역할에 특화된 AI로 만들 수 있습니다.

Q: 어떻게 작성하나요?

A: 평범한 한국어로 작성하면 됩니다! 프로그래밍 지식 불필요.

✍️ 실전 예시

❌ 나쁜 지침

고객 응대를 잘 해줘.

→ 너무 모호함. 구체적인 방향 없음.

✅ 좋은 지침

당신은 OO전자 고객센터 상담사입니다. 역할: - 제품 문의, A/S 접수, 환불 요청에 대응 - 항상 친절하고 공손한 말투 사용 - "~습니다", "~입니다" 존댓말 필수 규칙: - 먼저 고객의 이름을 물어보고 인사 - 문제를 경청한 후 3가지 해결 방안 제시 - 즉시 해결 불가능한 경우 "담당 부서에 전달하겠습니다" 안내 - 절대 추측하지 말고, 확실하지 않으면 "확인 후 연락드리겠습니다" 답변 금지사항: - 욕설이나 비속어 사용 금지 - 개인정보(주민번호, 계좌번호) 수집 금지 - 경쟁사 제품 언급 금지

→ 구체적인 역할, 규칙, 금지사항 명시

💡 작성 팁

1. 역할 명확히

"당신은 ~입니다"로 시작

2. 말투 지정

존댓말/반말, 이모티콘 사용 여부 등

3. 예시 포함

"예: 고객님, 무엇을 도와드릴까요?"

4. 제약사항

하면 안 되는 것도 명시

🎓 실습 과제

나만의 GPT 지침 작성해보기:

  1. ChatGPT Plus 로그인
  2. 좌측 "Explore" → "Create a GPT" 클릭
  3. "Configure" 탭에서 Instructions 작성
  4. 위 예시를 참고하여 여러분의 업무에 맞는 지침 작성
  5. Preview에서 테스트해보기

대화 시작 템플릿

사용자가 쉽게 시작할 수 있도록 돕는 버튼

🎯 대화 시작 템플릿이란?

GPT를 처음 열었을 때 화면에 보이는 예시 질문 버튼들입니다. 사용자가 "뭘 물어볼까?" 고민하지 않도록 미리 준비해둔 질문 리스트입니다.

📝 초급자를 위한 설명

Q: 왜 필요한가요?

A: 빈 화면만 보면 사용자는 당황합니다. "이 GPT가 뭘 할 수 있지?"를 직접 보여주면 훨씬 사용하기 쉽습니다.

비유: 식당 메뉴판과 같습니다. 메뉴판 없이 "뭐 먹을래?" 하면 고민되지만, "추천 메뉴 3가지"를 보여주면 바로 선택할 수 있죠!

예시 1: 영어 회화 GPT

예시 2: 코딩 도우미 GPT

예시 3: 마케팅 GPT

💡 작성 팁

1. 구체적으로

❌ "도와줘" → ✅ "이력서 작성 도와줘"

2. 3~5개

너무 많으면 복잡, 적으면 선택지 부족

3. 이모티콘 활용

시각적으로 구분, 친근한 느낌

4. 핵심 기능 위주

GPT의 주요 용도를 대표하는 질문

🎓 실습 과제

나만의 대화 시작 템플릿 만들기:

  1. GPT Builder에서 "Configure" 탭 열기
  2. "Conversation starters" 섹션 찾기
  3. 여러분의 GPT가 가장 잘 하는 것 3~5가지 생각
  4. 각각을 짧은 질문 형태로 작성
  5. Preview에서 버튼 클릭해보며 테스트

지식 (Knowledge)

GPT에게 전문 지식을 가르치는 방법

🎯 지식(Knowledge)이란?

GPT에게 파일을 업로드하여 추가 정보를 학습시키는 기능입니다. 마치 교과서를 주는 것처럼, GPT가 해당 내용을 참고하여 답변합니다.

📝 초급자를 위한 설명

Q: 왜 필요한가요?

A: ChatGPT는 2023년까지의 일반적인 지식만 알고 있습니다. 하지만 여러분 회사의 제품 정보, 내부 규정, 최신 자료는 모릅니다. Knowledge로 이런 정보를 제공하면 GPT가 여러분의 전문가가 됩니다!

비유: 신입사원(GPT)에게 회사 업무 매뉴얼을 주는 것과 같습니다.

📂 업로드 가능한 파일

📄

문서

PDF, Word (.docx), PowerPoint (.pptx)

📊

데이터

Excel (.xlsx), CSV, JSON

📝

텍스트

TXT, Markdown (.md), 노션 페이지

✍️ 실전 예시

사례 1: 고객 상담 GPT

업로드 파일:

  • 제품 사용 설명서.pdf
  • 자주 묻는 질문 FAQ.docx
  • A/S 정책.pdf

결과: GPT가 제품 문의에 정확하게 답변!

사례 2: 법률 자문 GPT

업로드 파일:

  • 근로기준법.pdf
  • 판례 모음.docx
  • 회사 취업규칙.pdf

결과: 법률 질문에 근거 있는 답변 제공!

사례 3: 레시피 GPT

업로드 파일:

  • 전통 한식 레시피 100선.pdf
  • 식재료 영양 정보.xlsx
  • 조리 팁.txt

결과: 한식 요리법을 자세히 알려줌!

⚠️ 주의사항

1. 파일 개수 제한

최대 10개까지만 업로드 가능

2. 민감 정보 금지

개인정보, 기밀 문서는 업로드 금지

3. 파일 크기

너무 큰 파일은 일부만 읽힐 수 있음

4. 수정 어려움

업로드 후 수정은 재업로드 필요

🎓 실습 과제

Knowledge 업로드 실습:

  1. GPT Builder "Configure" 탭 → "Knowledge" 섹션
  2. "Upload files" 버튼 클릭
  3. 회사 자료나 개인 문서 1~3개 선택 (민감정보 제외)
  4. 업로드 완료 후 Preview에서 테스트
    예: "업로드한 문서에 대해 요약해줘"
  5. GPT가 파일 내용을 인식하는지 확인

기능 (Capabilities)

GPT에게 슈퍼파워를 부여하세요

🎯 기능(Capabilities)이란?

GPT가 텍스트 대화 이상의 일을 할 수 있게 해주는 내장 도구들입니다. 켜고 끄는 스위치처럼 간단하게 활성화할 수 있습니다.

🔧 사용 가능한 기능

🌐

Web Browsing (웹 브라우징)

무엇을 하나요?
인터넷을 검색하여 최신 정보를 찾아옵니다.

언제 사용하나요?
뉴스, 날씨, 주식, 최신 트렌드 등 실시간 정보 필요 시

예시: "오늘 서울 날씨 알려줘" → GPT가 실시간 검색!
💻

Code Interpreter (코드 실행)

무엇을 하나요?
파이썬 코드를 직접 실행하고, 파일을 분석하거나 그래프를 그립니다.

언제 사용하나요?
데이터 분석, 엑셀 처리, 이미지 편집, 복잡한 계산

예시: CSV 파일 업로드 → "매출 추이 그래프 그려줘"
🎨

이미지 생성 (GPT-4o)

무엇을 하나요?
텍스트 설명을 입력하면 AI가 이미지를 그립니다. 2025년부터 GPT-4o가 기본 엔진입니다.

언제 사용하나요?
로고, 일러스트, 프레젠테이션 이미지, 스타일 변환 필요 시

예시: "귀여운 고양이가 우주를 여행하는 그림 그려줘" 또는 "이 사진을 지브리 스타일로 변환해줘"
👁️

Vision (이미지 인식)

무엇을 하나요?
사용자가 올린 이미지를 보고 분석합니다.

언제 사용하나요?
사진 설명, 문서 OCR, 다이어그램 해석

예시: 음식 사진 업로드 → "이 요리 레시피 알려줘"

✍️ 실전 활용 예시

조합 1: 웹 + Code Interpreter

액션 (Actions)

GPT를 외부 세상과 연결하세요

🎯 액션(Actions)이란?

GPT가 외부 서비스의 API를 호출하여 데이터를 가져오거나 작업을 수행하게 만드는 고급 기능입니다. 마치 GPT에게 전화번호부를 주는 것처럼, 필요한 서비스에 직접 연락하게 합니다.

📝 초급자를 위한 설명

Q: API가 뭔가요?

A: Application Programming Interface의 약자로, 프로그램끼리 정보를 주고받는 창구라고 생각하면 됩니다. 예를 들어 날씨 API는 "서울 날씨 알려줘"라고 요청하면 실시간 날씨 데이터를 돌려줍니다.

Q: 왜 필요한가요?

A: Knowledge는 "과거 자료"만 업로드 가능하지만, Actions는 "실시간 데이터"나 "실제 작업"이 가능합니다.

비유: Knowledge = 책, Actions = 전화기

→

사례 2: 재고 관리 GPT

사용자: "A제품 재고 확인"
→
GPT가 회사 DB API 호출
→
재고 데이터 조회
→
답변: "A제품 남은 수량: 150개"

사례 3: 예약 GPT

사용자: "내일 2시 회의실 예약"

🔧 설정 방법 (간단 버전)

Step 1: API 찾기

사용할 외부 서비스의 API 문서 확인

예: OpenWeather API

Step 2: OpenAPI 스펙 작성

API를 GPT가 이해할 수 있는 형식(JSON)으로 정의

{ "openapi": "3.0.0", "info": { "title": "날씨 조회", "version": "1.0.0" }, "servers": [ {"url": "https://api.openweathermap.org"} ], "paths": { "/weather": { "get": { "operationId": "getWeather", "parameters": [ {"name": "q", "in": "query", "description": "도시 이름"} ] } } } }

Step 3: GPT에 등록

GPT Builder의 "Actions" → "Create new action" → 위 JSON 붙여넣기

Step 4: 테스트

Preview에서 실제로 API가 호출되는지 확인

⚠️ 주의사항

1. API 키 보안

절대 공개 GPT에 API 키 노출 금지

3. 비용 발생

4. 기술 난이도

OpenAPI 스펙 작성은 다소 복잡함

GPTs와 SerpAPI 연동

실시간 웹 검색 기능을 GPT에 추가하여 최신 정보 제공

🔍 SerpAPI란?

Google, Naver, YouTube 등 다양한 검색엔진의 결과를 API 호출로 JSON 형태로 제공하는 서비스

✅ 복잡한 웹 크롤링 불필요
✅ reCAPTCHA 우회, IP차단 위험 없음
✅ 안정적인 데이터 수집
✅ 다양한 검색엔진 지원

🛠️ 사용 방법

1. SerpAPI 계정 생성 및 API 키 발급

월 100회 무료 사용 가능

2. 엔드포인트 이해
GET https://serpapi.com/search?
engine=google&
q=날씨 서울&
api_key=YOUR_API_KEY

→ 서울 날씨에 대한 구글 검색 결과 JSON 반환

3. GPT Action 설정

GPT Builder의 Action란에 SerpAPI 호출 스키마 정의

operationId: "SearchNews"
parameters: { q: "검색어" }
url: "https://serpapi.com/search"
4. 테스트

사용자: "최신 AI 업계 뉴스 알려줘" → GPT가 SerpAPI 호출 → JSON 파싱 → 요약 답변

SerpAPI → 네이버 검색 API →

네이버 뉴스레터 자동화 봇

네이버 콘텐츠 수집 → GPT 요약 → 자동 발행/배포하는 뉴스레터 시스템

📰 워크플로우

1

콘텐츠 수집

방법 1: 네이버 검색 API / SerpAPI로 최신 뉴스 검색

방법 2: 네이버 뉴스 RSS 피드 구독

방법 3: 네이버 블로그 RSS 모니터링

Make의 RSS 모듈로 주기적 확인 (예: 매일 오전 8시)
2

콘텐츠 가공

수집한 원문을 GPT에게 전달

프롬프트 템플릿:
"다음 3개 뉴스를 요약하고, 뉴스레터 형식으로 편집해줘.
형식: 제목, 2줄 요약, 출처 링크"

→ GPT가 깔끔한 뉴스레터 포맷으로 작성

3

발행/배포

이메일 발송: Make의 Email 모듈로 구독자에게 자동 전송

네이버 카페 게시: HTTP 모듈로 카페 글쓰기 (OAuth 인증 필요)

블로그 포스팅: 네이버 블로그 API 활용

💼 예시 시나리오

목표: 매일 아침 8시, IT 업계 주요 뉴스 3개를 이메일로 받기

1️⃣ Make 스케줄러: 매일 오전 7시 실행
2️⃣ SerpAPI: "IT 뉴스" 네이버 검색 → 상위 5개 기사 수집
3️⃣ OpenAI 모듈: 5개 기사 중 중요한 3개 선정 + 요약
4️⃣ Gmail 모듈: 요약본을 내 이메일로 발송
